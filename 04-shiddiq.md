# Rekayasa Perangkat Lunak (RPL) – Pembahasan Terpadu

## 1. Konsep Dasar: SDLC

**SDLC (Software Development Life Cycle)** adalah proses terstruktur dalam pengembangan perangkat lunak, mulai dari perencanaan hingga pemeliharaan.

### Model SDLC:

### a. Waterfall

* Bersifat **linear/berurutan**
* Tahapan:

  1. Analisis kebutuhan
  2. Desain sistem
  3. Implementasi (coding)
  4. Testing
  5. Maintenance

### b. Agile (Scrum & XP)

* Bersifat **iteratif & fleksibel**
* Fokus pada perubahan cepat dan feedback pengguna

**Scrum:**

* Product Backlog
* Sprint
* Sprint Backlog
* Daily Scrum

---

## 2. Tahapan Pengembangan: PADI

PADI menggambarkan alur utama dalam pengembangan sistem:

* **Perencanaan**
  → Menentukan tujuan, ruang lingkup, dan kebutuhan awal proyek

* **Analisis**
  → Mengidentifikasi masalah serta kebutuhan sistem secara detail

* **Desain**
  → Merancang sistem (arsitektur, database, tampilan)

* **Implementasi**
  → Membangun sistem (coding) dan mulai digunakan

  PADI membantu memahami **alur dari ide sampai sistem jadi**

---

## 3. Kelayakan Proyek (PREP)

Digunakan untuk menilai apakah proyek layak dikerjakan:

* **People** → SDM kompeten
* **Resource** → teknologi tersedia
* **Economy** → biaya dan manfaat
* **Process** → proses kerja jelas

---

## 4. Estimasi Proyek Software

### a. SLOC

* Mengukur ukuran software dari jumlah baris kode

### b. COCOMO

* Mengestimasi:

  * Biaya
  * Waktu
  * Tenaga kerja

---

## 5. Pengujian Software (Testing)

* **Blackbox Testing** → uji fungsi
* **Whitebox Testing** → uji kode
* **Graybox Testing** → kombinasi

---

## 6. Struktur Tim Pengembangan

* Project Manager (PM)
* Developer
* QA (Quality Assurance)
* Designer (UI/UX)

---

## 7. Sifat Iteratif Pengembangan

* Pengembangan software bisa **berulang (iteratif)**
* Tidak selalu linear
* Sangat cocok dengan Agile
