 
  📑 Dokumen Rangkuman: Rekayasa Perangkat Lunak
1. Metodologi Pengembangan (SDLC)                          
Bagian ini mengatur strategi besar tim dalam menjalankan proyek.           

1.1. Model Waterfall: Gaya kerja linear/berurutan. Tahap selanjutnya baru bisa jalan kalau tahap sebelumnya sudah beres total.  

1.2. Model Agile: Gaya kerja lincah dan berputar. Fokus pada rilis fitur kecil secara cepat dan terus menerus.   

1.2.1. Scrum: Kerangka kerja untuk mengatur kolaborasi tim.         

1.2.2. Sprint: Siklus waktu singkat (1-4 minggu) untuk menyelesaikan target fitur.      

1.2.3. XP (Extreme Programming): Praktik pengembangan yang sangat fokus pada kualitas kode murni.            
                                                                     
2. Alur Kerja Produksi (Siklus P-A-D-I & U-D-M)
Tahapan teknis dari awal ide muncul sampai aplikasi bisa dipakai dan dirawat.
                                                                                                                                                                
2.1. Planning (P): Nyusun rencana awal tentang apa yang mau dibuat, siapa timnya, dan berapa estimasi biayanya.

2.2. Analysis (A): Proses membedah kebutuhan user dan fitur apa saja yang wajib ada di dalam sistem.

2.3. Design (D): Membuat cetak biru aplikasi, mulai dari rancangan database, alur sistem, hingga tampilan (UI).

2.4. Implementation (I): Tahap eksekusi utama di mana programmer mulai menulis kode (coding).

2.5. Unit Testing (U): Melakukan tes pada setiap bagian kecil kode untuk memastikan tidak ada bug yang tersisa.

2.6. Deployment (D): Merilis aplikasi ke server atau lingkungan produksi agar bisa mulai diakses oleh pengguna.

2.7. Maintenance (M): Melakukan perawatan rutin, memperbaiki error yang baru muncul, dan memberikan pembaruan.
                                                                                                         
3. Kontrol Kualitas & Estimasi Proyek
Alat ukur untuk memastikan aplikasi bagus dan proyek tidak rugi waktu/biaya.
                                                                                                                                    
3.1. Strategi Testing (Cek Error):

3.1.1. White Box: Cek jeroan kode sampai ke alur logikanya secara mendalam.

3.1.2. Black Box: Cek fungsi luar (input-output) hanya dari kacamata pengguna awam.

3.2. Metrik Pengukuran (Hitung Skala):

3.2.1. COCOMO: Rumus untuk memprediksi butuh berapa orang dan berapa lama proyek bisa selesai.

3.2.2. LOC (Line of Code): Menghitung skala aplikasi berdasarkan total baris kode yang ditulis.

3.2.3. Function Point (FP): Menghitung nilai aplikasi berdasarkan seberapa banyak fitur yang berguna bagi user.
                                                                                 
4. Fondasi Teknologi & Integrasi
Isi "dapur" yang menghubungkan antar sistem dan data.

4.1. Integrasi Sistem: Menggunakan API sebagai jembatan komunikasi antar aplikasi berbeda.

4.2. Format Data: Standar pengiriman data yang digunakan adalah JSON atau XML.

4.3. Main Stack: Senjata utama pengembangan menggunakan PHP, SQL, HTML, JS, dan CMS.
