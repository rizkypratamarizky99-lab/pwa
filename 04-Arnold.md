
# Rangkuman Software Development Life Cycle (SDLC)

## 1. Siklus Hidup Pengembangan Perangkat Lunak (SDLC)

SDLC adalah proses terstruktur yang digunakan untuk merancang, membangun, menguji, dan memelihara perangkat lunak agar sesuai dengan kebutuhan pengguna.

### Tahapan SDLC:
1. **Planning (Perencanaan)**  
   Tahap awal untuk menentukan tujuan proyek, ruang lingkup, serta sumber daya yang dibutuhkan.

2. **Analysis (Analisis Kebutuhan)**  
   Mengidentifikasi kebutuhan pengguna dan sistem yang akan dibuat.

3. **Design (Perancangan Sistem)**  
   Membuat desain arsitektur sistem, database, dan tampilan antarmuka.

4. **Implementation (Coding / Pengembangan)**  
   Tahap pengkodean berdasarkan desain yang telah dibuat.

5. **Testing (Pengujian)**  
   Memastikan sistem berjalan sesuai kebutuhan dan bebas dari error.

6. **Maintenance (Pemeliharaan)**  
   Perbaikan bug, update fitur, dan peningkatan sistem setelah digunakan.

---

### Model Pengembangan:
Model pengembangan berurutan (linear), di mana setiap tahap harus selesai sebelum lanjut ke tahap berikutnya. Cocok untuk proyek dengan kebutuhan yang sudah jelas.
Ada tiga pendekatan utama yang dibahas di papan tulis untuk menentukan alur kerja tim:
* **Waterfall:** Model kaku dan berurutan. Cocok untuk proyek yang persyaratannya sudah jelas sejak awal.

* **Agile:** Fokus pada kecepatan dan perubahan.
* **Scrum:** Kerja dalam siklus pendek (**Sprint**).
* **XP (Extreme Programming):** Fokus pada kualitas teknis (koding).
* **Spiral:** Model melingkar yang mengutamakan **Analisis Risiko**. Setiap putaran berarti evaluasi ulang agar proyek tidak gagal di tengah jalan.

---

## 2. Pengujian Perangkat Lunak (Software Testing)

Testing bertujuan untuk memastikan sistem berjalan dengan benar, aman, dan sesuai kebutuhan.

### Jenis Testing:

- **Black Box Testing**  
  Menguji fungsi sistem tanpa melihat kode program. Fokus pada input dan output.

- **White Box Testing**  
  Menguji struktur internal kode, seperti logika dan alur program.

- **Gray Box Testing**  
  Kombinasi antara Black Box dan White Box, tester memiliki sedikit pengetahuan tentang sistem.

- **Penetration Testing (PenTest)**  
  Pengujian keamanan untuk menemukan celah yang bisa diserang oleh hacker.

---

## 3. Estimasi dan Metrik Software

Digunakan untuk memperkirakan ukuran, biaya, dan waktu pengerjaan proyek.

- **LOC (Line of Code)**  
  Mengukur ukuran software berdasarkan jumlah baris kode.

- **SLOC (Source Lines of Code)**  
  Versi lebih spesifik dari LOC, hanya menghitung kode asli.

- **Function Point (FP)**  
  Mengukur berdasarkan jumlah fungsi yang diberikan kepada pengguna.

- **COCOMO (Constructive Cost Model)**  
  Model matematis untuk menghitung biaya, waktu, dan tenaga kerja proyek.

---

## 4. Teknologi dan Komunikasi Data

### 🔹 API (Application Programming Interface)
API adalah penghubung antar sistem yang memungkinkan aplikasi saling berkomunikasi.

### 🔹 Format Data
- **JSON** → ringan dan mudah digunakan
- **XML** → lebih kompleks dan terstruktur

### 🔹 Tech Stack
- **PHP** → backend
- **HTML** → struktur halaman web
- **JavaScript** → interaksi dan logika frontend
- **SQL** → pengelolaan database

### 🔹 CMS (Content Management System)
Sistem yang digunakan untuk mengelola konten website tanpa harus coding dari awal.

---

## 5. Manajemen Proyek & Struktur Tim

### Tahapan Manajemen:
- **Monitoring & Control**  
  Mengawasi perkembangan proyek agar tetap sesuai rencana.

- **Analisis & Desain**  
  Menentukan kebutuhan dan rancangan sistem sebelum coding.

### Peran dalam Tim:
- **Project Manager (PM)** → mengatur jalannya proyek
- **Developer** → membuat sistem
- **QA / Tester** → menguji sistem
- **Designer** → membuat tampilan UI/UX
- **Analyst** → menganalisis kebutuhan sistem

---

## 6. Konsep Tambahan

- **Input → Process → Output**  
  Dasar cara kerja sistem: data masuk → diproses → menghasilkan output.

- **Manual vs Otomatis**  
  Sistem manual membutuhkan manusia, sedangkan otomatis menggunakan sistem komputer.

- **Database (SQL)**  
  Tempat menyimpan dan mengelola data.

- **Agile Mindset**
  - Iterasi cepat
  - Kolaborasi tim
  - Mudah beradaptasi terhadap perubahan

---

## 📌 Kesimpulan

Software Development Life Cycle (SDLC) adalah kerangka kerja penting dalam pengembangan perangkat lunak yang mencakup:
- Perencanaan hingga pemeliharaan
- Metode pengembangan (Waterfall, Agile, Scrum, XP)
- Pengujian sistem
- Estimasi biaya dan waktu
- Teknologi yang digunakan
- Manajemen tim dan proyek
